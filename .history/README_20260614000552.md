# 🚀 Brazil Football Hub — Deployment and GitHub Sync Guide

This secondary guide provides a quick-start checklist specifically for pushing your code to **GitHub** and integrating it with **Netlify** to achieve automatic build triggers on every commit.

---

## 📋 Pre-Flight Checklist

Before you begin, make sure you have:
1. A **[GitHub Account](https://github.com/)** (Free)
2. A **[Netlify Account](https://netlify.com/)** (Free, using email or GitHub login)
3. **[Node.js](https://nodejs.org)** installed locally (for testing changes)
4. **Git** installed on your computer.

---

## 🛠️ Step 1: Uploading to GitHub

Follow these command-line instructions to push your project files to a new GitHub repository:

1. **Sign in to GitHub** and click the green **New** button under your repository panel.
2. Provide a name, e.g., `brazil-football-fanhub` (keep other options default: do *not* add a README, `.gitignore`, or License as the code already includes these configured).
3. Open your terminal at the project directory root `/` and enter:

```bash
# Initialize git inside the directory
git init

# Add all files (the local .gitignore ensures node_modules, dist, etc. are skipped)
git add .

# Create the first build checkpoint
git commit -m "feat: complete pristine brazil football hub launch"

# Rename default branch to main
git branch -M main

# Link this local repository to your remote GitHub page
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPOSITORY_NAME.git

# Push the code to GitHub's cloud main branch
git push -u origin main
```

*Change `YOUR_USERNAME` and `YOUR_REPOSITORY_NAME` to match your actual GitHub handle and repository name.*

---

## 🔋 Step 2: Connect to Netlify (Continuous Delivery)

Connecting Netlify directly to your repository is highly recommended. Every time you push a minor text change or update images on GitHub, Netlify automatically builds and redeploys the site in seconds!

1. Open your **[Netlify Dashboard](https://app.netlify.com/)** and click **"Add new site"** > **"Import from Git"**.
2. Select **GitHub** as your Git provider. If prompted, authorize Netlify to access your repositories.
3. Search for your repository `brazil-football-fanhub` and click it.
4. Netlify will scan the files and automatically detect standard Vite presets. Verify the forms match this:
   * **Branch to deploy**: `main`
   * **Build command**: `npm run build`
   * **Publish directory**: `dist`
5. Click **"Deploy site"**. 

*That's it! Netlify is now building your site. The first deployment usually takes under 45 seconds. Once finished, Netlify will generate a beautiful custom preview link.*

## 🌐 Customized Domain Name (Optional)

If you would like to map your own custom domain (e.g. `www.yourname.com`) to your Netlify page:
1. In the Netlify console, go to **Site settings** > **Domain management**.
2. Click **Add custom domain**.
3. Type in your domain name and follow the standard instructions to add the CNAME record into your DNS registrar (Namecheap, GoDaddy, Cloudflare, etc.).
4. Netlify automatically provisions a **100% free SSL Let's Encrypt Certificate** for your domain for secure HTTPS connections.

## 📦 Local Maintenance Checklist

If you ever wish to edit the files locally in the future:
* **To run the live workspace locally**: `npm run dev`
* **To ensure code syntax has zero warnings**: `npm run lint`
* **To test manual build locally**: `npm run build` (outputs build folder `/dist`)
